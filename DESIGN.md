# დიზაინის დოკუმენტაცია (Design Decisions) — Report Scheduler

ეს დოკუმენტი აჯამებს "Report Scheduler" ვებ-პროტოტიპის ვიზუალური ენის, არქიტექტურისა და კომპონენტების დიზაინის ძირითად გადაწყვეტილებებს.

## 1. ვიზუალური იდენტობა (Design System & System Tokens)

აპლიკაცია ეფუძნება **თანამედროვე მინიმალიზმს** (Modern Minimalism) და შექმნილია Windows Desktop აპლიკაციების საუკეთესო პრაქტიკების შესაბამისად:

* **ცენტრალური ტოკენები (`src/theme.ts` და CSS Variables):**
  ყველა დიზაინ-კონსტანტა (Accent ფერები, შრიფტები, ზომები, დაშორებები) გატანილია ერთ ცენტრალიზებულ `THEME_TOKENS` ფაილში და დაკავშირებულია Tailwind-ის CSS ცვლადებთან (`src/index.css`), რაც ამარტივებს მათ მართვასა და კონსისტენტურობას.
* **ძირითადი აქცენტი (Primary Blue):** `#185FA5` — გამოიყენება ძირითადი ქმედებებისთვის, აქტიური ფილტრებისთვის, ღილაკებისა და ფოკუსის მდგომარეობებისთვის.
* **დაგლუვება (Rounded Corners):**
  * მართვის პანელები, ფორმის ველები და ღილაკები: `8px` (`rounded-lg`).
  * კონტეინერები, ბარათები და მოდალური ფანჯრები: `12px` (`rounded-xl`).
* **თხელი ხაზები (Tonal Layering):** მძიმე ჩრდილების ნაცვლად გამოყენებულია თხელი hairline ხაზები (`0.5-1px` border-ები) და რბილი კონტრასტული კონტეინერები სიღრმის გამოსახატად.

## 2. ფერთა პალიტრა და თემები (Theme Toggle & States)

პროტოტიპს აქვს სრულად ფუნქციური **Light / Dark** თემების გადამრთველი:

| ელემენტი / მდგომარეობა | Light მნიშვნელობა | Dark მნიშვნელობა |
| :--- | :--- | :--- |
| **სამუშაო ფონი (`--bg`)** | `#F3F4F6` (სუფთა ნაცრისფერი) | `#0B0F19` (ღრმა ლურჯი-მუქი) |
| **ბარათების ფონი (`--card`)** | `#FFFFFF` (თეთრი) | `#111827` (მუქი ნაცრისფერი) |
| **საზღვრები (`--border`)** | `#E5E7EB` (ნაზი ნაცრისფერი) | `#1F2937` (თხელი მუქი ხაზი) |
| **ტექსტი (`--text`)** | `#111827` (Charcoal შავი) | `#F9FAFB` (თოვლისფერი თეთრი) |

### სტატუსის ფერები და ბეჯები:
* **წარმატება / SQL Server ბეჯი:** `SQL Server` იყენებს ღია ცისფერ ტონალობას (`#E0F2FE` ფონი, `#0369A1` ტექსტი), ხოლო `წარმატების` სტატუსი გამოსახულია მწვანე ფერით (`#059669`).
* **MySQL ბეჯი:** იყენებს რბილ ფირუზისფერ ტონალობას (`#F0FDFA` ფონი, `#0F766E` ტექსტი).
* **შეცდომა:** წითელი აქცენტი (`#DC2626`) `ჯარიმების შეჯამების` დაფიქსირებულ შეცდომაზე.

## 3. ტიპოგრაფია (Typography Pairings)

* **ძირითადი ინტერფეისი:** Google Fonts-ის ელეგანტური **Inter** შრიფტი, რომელიც უზრუნველყოფს იდეალურ წაკითხვადობას მაღალი სიმჭიდროვის მქონე Desktop ცხრილებსა და მოდალურ ველებში.
* **მონაცემები და კოდი:** **JetBrains Mono** მონოსპეის შრიფტი, რომელიც გამოიყენება SQL მოთხოვნებისა და სისტემური ლოგების სექციისთვის ("ლოგი სტატუს კონსოლი").
* **წინადადების რეგისტრი (Sentence Case):** ყველა ქართული წარწერა დაცულია ლიტერატურული ენის ნორმების შესაბამისი ფორმატით, რაც პროფესიონალურად გამოიყურება.

---

## 4. მომხმარებლის სამუშაო ნაკადი (UX User Flows)

1. **Dashboard-ის ეკრანი:** გვაწვდის ინფორმაციას რეალურ დროში დათვლილი მეტრიკებით (აქტიური სამუშაოები, დღეს გაშვებული და შეცდომის მქონე ჯობები).
2. **ახალი ანგარიშის 2-Step Wizard:**
   * **Step 1 (კავშირი):** სახელი, გასავალი ფოლდერი (შემოწმების სიმულატორით), სერვერის პარამეტრები და "კავშირის ტესტირების" დინამიკური ინლაინ ვალიდაცია.
   * **Step 2 (განრიგი):** გაშვების სიხშირის პერიოდული კონტროლი, დროის მულტი-პილების რეალური დამატება/წაშლა და მრავალ-მეილიანი მიმღებების ჩიპები.
3. **დეტალური ისტორია & ლოგები:** თითოეული ანგარიშის ბოლო 5 გაშვების რეალური ისტორია, სტატუსებით, ხანგრძლივობითა და სისტემური ტერმინალის ლოგის პანელით.
